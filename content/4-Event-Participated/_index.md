# EVENTS PARTICIPATED

⚠️ **Note**: The information below documents actual events I participated in during my AWS FCJ internship from September to December 2025. Each event provided valuable learning experiences and opportunities to connect with the AWS community.

During my internship, I participated in three significant AWS-related events. Each one was a memorable experience that provided new, interesting, and useful knowledge, along with wonderful networking moments with cloud professionals.

---

## Event 1: Vietnam Cloud Day 2025 - Ho Chi Minh City Connect Edition

**Event Name:** Vietnam Cloud Day 2025 - Ho Chi Minh City Connect Edition for Builders

**Date & Time:** 09:00 AM, September 18, 2025

**Location:** Ho Chi Minh City, Vietnam

**Role:** Attendee

### Event Overview

Vietnam Cloud Day 2025 was a premier cloud computing conference bringing together builders, developers, and cloud enthusiasts across Vietnam. The event focused on the latest AWS services, cloud architecture best practices, and real-world use cases from Vietnamese enterprises adopting cloud technologies.

### Key Topics Covered

- **AWS Security Services Updates**: Latest features in AWS IoT Core security, certificate management, and device authentication
- **Serverless Architecture Patterns**: Building scalable applications with Lambda, API Gateway, and EventBridge
- **IoT Security Best Practices**: Implementing PKI infrastructure for IoT device fleet management
- **Cloud Architecture Design**: Multi-tier security architecture for enterprise applications
- **AWS Well-Architected Framework**: Security pillar and operational excellence principles

### Sessions Attended

1. **Securing IoT Devices at Scale** - AWS Solutions Architect
   - Certificate-based authentication strategies
   - AWS IoT Core policies and thing groups
   - Integration with AWS Certificate Manager Private CA

2. **Building Serverless Applications** - AWS Developer Advocate
   - Lambda function optimization techniques
   - EventBridge for event-driven architecture
   - DynamoDB best practices for high-throughput applications

3. **Customer Success Stories** - Vietnamese Enterprise Panel
   - Real-world IoT deployments in manufacturing and logistics
   - Cost optimization strategies for production workloads
   - DevOps practices for cloud-native applications

### Outcomes and Value Gained

- **Technical Knowledge**: Deepened understanding of AWS security services directly applicable to my IoT Security Infrastructure project, particularly around PKI/TLS certificate management
- **Architecture Insights**: Learned best practices for designing 3-tier certificate authority hierarchies and automating certificate lifecycle management
- **Networking**: Connected with 20+ AWS professionals, including Solutions Architects and fellow developers working on similar IoT security challenges
- **Project Validation**: Received feedback from AWS experts on my project architecture, validating the serverless approach and suggesting improvements for scalability
- **Community Engagement**: Joined AWS Vietnam User Group and connected with mentors for ongoing technical guidance

### Impact on Internship Project

This event was instrumental in shaping my project's technical direction:
- Refined PKI architecture based on AWS recommended patterns
- Identified AWS services (ACM Private CA, Lambda, DynamoDB) as core components
- Learned security best practices that influenced project design decisions
- Gained confidence in serverless architecture approach

**Event Link:** [Vietnam Cloud Day](https://vietnamcloudday.vn/)

## Event 3: AWS Cloud Mastery Series #1

**Event Name:** AWS Cloud Mastery Series #1 - Advanced Security Architecture

**Date & Time:** 10:00 AM, November 13, 2025

**Location:** Virtual Event (AWS Webinar)

**Role:** Attendee

### Event Overview

Part of AWS's ongoing education series, this webinar focused on advanced security architecture patterns for enterprise AWS environments. It covered multi-account strategies, AWS Organizations best practices, and security automation at scale.

### Key Topics Covered

- **Multi-Account Security Architecture**: Designing secure, scalable AWS account structures
- **AWS Organizations Best Practices**: Implementing Service Control Policies (SCPs) and organizational units
- **Security Automation**: Using AWS Security Hub, GuardDuty, and Config for automated compliance
- **Identity and Access Management**: Advanced IAM patterns including ABAC and permission boundaries
- **Incident Response**: Building automated incident response workflows with Lambda and Step Functions

### Sessions Attended

1. **Multi-Account Strategy for Startups** - AWS Solutions Architect
   - Account segregation patterns (dev/test/prod)
   - Centralized logging and monitoring architecture
   - Cross-account IAM role assumptions

2. **Security Automation at Scale** - AWS Security Specialist
   - Automated compliance checking with AWS Config Rules
   - GuardDuty for threat detection and response
   - Security Hub for centralized security findings

3. **IAM Best Practices and Common Pitfalls**
   - Principle of least privilege implementation
   - Using permission boundaries effectively
   - IAM policy evaluation logic deep dive

### Outcomes and Value Gained

- **Architecture Refinement**: Applied multi-account concepts to plan scalability path for IoT project
- **Security Posture**: Enhanced project security by implementing automated compliance checks
- **IAM Expertise**: Improved IAM policy design with more granular, least-privilege permissions
- **Monitoring Strategy**: Learned to set up comprehensive security monitoring and alerting
- **Career Development**: Gained knowledge applicable to AWS Security Specialty certification path

### Impact on Internship Work

- **Project Enhancement**: Added security monitoring dashboard to IoT platform
- **IAM Policies**: Refined Lambda execution roles with more precise permissions
- **Documentation**: Created security architecture documentation following AWS best practices
- **Future Planning**: Designed scalability roadmap for multi-account deployment
